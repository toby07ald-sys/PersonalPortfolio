Testing & Quality Assurance

NCTX System Unit Testing

Description: A manual testing regime for the Bus Ticketing CLI to ensure data saved to CSVs was accurate and retrievable.

Methodology: Boundary Value Analysis & Input Validation.

Key Tests:

  • Input Validation: Intentionally entered invalid data types (e.g., text into age fields) to verify the program handled errors without crashing.
  • Boundary Checks: Tested the exact age limits for ticket types (e.g., inputting '17' vs '18') to verify restriction logic.
  • CSV Integrity: Verified that user data persisted correctly after a system restart.

Context: Python Coursework (Verification Phase).
